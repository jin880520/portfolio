본 문서는 포트폴리오 공개를 위해 실제 서비스의 민감정보(계정, 비밀번호, 키/토큰, 실도메인/IP, 서버 경로, 고객 고유명 등)를 익명화/일반화했습니다. 기능 흐름과 운영 관점의 설명만 남기고, 보안에 영향을 줄 수 있는 세부는 제거했습니다.

참고사항 및 안내
설명

본 인수인계서는 주요 기능과 전체 흐름 중심으로 구성되어 있으며
단순 입력 위주의 페이지나 구조가 단순한 화면, 또는 화면 내 설명이 충분한 항목은 생략했습니다.

테스트/데모 데이터는 공개용 샘플 계정으로 대체합니다.

  • ID: demo_owner / PW: ********
  • ID: demo_admin / PW: ********
  • ID: demo_site_a / PW: ********
※ 실제 운영 환경에서는 비밀번호를 문서에 남기지 않고, 별도 비밀관리(Password manager / Vault 등)를 사용합니다.
메인 관리자 - 승인
설명

메인 관리자에서 관리자 회원 관리 페이지에서 가입 후 “심사중”인 관리자 계정을 승인/배정합니다.

승인 시 “유료 활성 상태”와 유사한 권한이 부여되며, 운영 정책에 따라 만료일(ms_expire)이 무제한/기간제 형태로 관리될 수 있습니다.

승인(또는 결제 완료) 시 아래와 같은 데이터가 생성됩니다.

(예시) 워크스페이스/사이트 매핑 테이블
manager_workspaces (구현별 명칭은 상이할 수 있음)

학습 데이터용 per-site 테이블이 생성될 수 있습니다(패턴만 공개).

training_qna_{site}
training_docs_{site}
site_menus_{site}
예시: training_qna_site_a

정적 리소스(이미지/CSS)는 사이트/관리자 단위로 자동 생성될 수 있습니다(경로는 일반화).

  • <APP_STATIC_ROOT>/img/<admin_or_site_key>/ (예: 로고/배경 등)
  • <APP_STATIC_ROOT>/css/main/<admin_or_site_key>/ (예: list/main/sub 스타일)
※ 실제 경로(/opt/...) 및 파일명 규칙은 공개용에서 마스킹했습니다.
메인 관리자 - 사이트 배정
설명

사이트 배정 시 기존 승인된 상위 관리자(서브-상위)의 하위 관리자(서브-하위)로 등록합니다.

승인된 사이트 목록에서 선택 후 사이트 배정을 실행하면 해당 사이트에 대한 운영 권한이 부여됩니다.

권한 구조 예시: Owner(메인) → Admin(서브-상위) → Admin(서브-하위)
사이트(관리자)별 커스텀
커스텀
설명

사이트 커스텀 시 기본 템플릿을 복사하여 사이트 전용 스타일로 확장합니다.

기본 CSS 템플릿(예시)

  • <APP_STATIC_ROOT>/css/main/template_default
  • <APP_STATIC_ROOT>/css/main/template_variant_a

복사 시 아래 경로에 <site_key> 전용 폴더를 생성하고 파일을 복사/수정합니다.

  • <APP_STATIC_ROOT>/css/main/<site_key>/
  • <site_key>_list.css
  • <site_key>_main.css
  • <site_key>_sub.css

배경 영상 파일 또한 사이트 단위로 관리될 수 있습니다.

  • <APP_STATIC_ROOT>/img/<site_key>/
  • <site_key>_bg.mp4
  • <site_key>_m_bg.mp4
※ 실제 운영 환경에서는 파일 업로드/배포 파이프라인(CDN, 빌드/릴리즈 등)과 연동될 수 있습니다.
서브 관리자 - 사이트 메뉴 및 페이지 추가
설명

서브 관리자는 내사이트 관리 → 메뉴 설정에서 메뉴 추가를 통해 사이트 메뉴 및 페이지를 구성할 수 있습니다.

  • 메뉴명, 아이콘, 정렬 순서를 입력
  • PC / 모바일 선택 시 선택한 환경에서만 노출

페이지 기능 타입(예시)

  • 게시판 - 대댓글 지원 일반 게시판
  • 영상 - 영상 게시판
  • 챗봇 - 내부 채팅 페이지
  • 통화연결 - 모바일 실제 통화(PC는 안내)
  • 링크 - 외부 링크 새 탭 연결
  • 뷰 - 관리자 입력 HTML을 렌더링하는 뷰 페이지

추가한 메뉴는 사용자 사이트 메뉴 및 관리자용 “챗봇 사이트” 메뉴 등에 자동 반영될 수 있습니다.

서브 관리자 - 내 챗봇 관리 - 챗봇 설정(핵심)
설명

서브 관리자는 내 챗봇 관리 → 챗봇 설정에서 워크스페이스(사이트)별 주요 환경을 관리합니다.

도메인 허용 목록

  • 사이트가 없는 경우(테스트/기본 운영): example.com, admin.example.com
  • 사이트가 있는 경우: 해당 사이트 도메인을 등록 (예: site-a.example.com)
  • 여러 개일 경우 쉼표(,)로 구분 (예: site-a.example.com, site-b.example.com)
  • 입력된 도메인은 워크스페이스 식별자 및 연동 자격증명(키/토큰 등)과 함께 검증됩니다. (자격증명은 문서 비공개)
실 IP/도메인 예시는 공개용에서 제거했습니다.
문서 예시 IP가 필요할 경우 203.0.113.0/24 같은 “문서용 예시 대역”을 사용합니다.

시스템 프롬프트

  • AI 대화 시 적용되는 기본 시스템 프롬프트
  • 답변 범위/톤/정책을 제어
  • 입력 내용이 많아질수록 토큰 사용량 증가 가능

이미지(OCR/비전) 프롬프트

  • 이미지 기반 분석 기능 사용 시 적용되는 프롬프트
  • 이미지 인식 규칙/금칙어/출력 형식 등을 정의
  • 서비스 목적에 맞게 최소한으로 유지하는 것을 권장
